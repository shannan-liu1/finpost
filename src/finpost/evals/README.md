# finpost.evals

Exact-answer evaluation primitive for Phase 1 post-training experiments.
Evaluates one or more checkpoints on one or more benchmark sources using
greedy decoding and deterministic answer parsing, then writes five output
files per run.

PRD: [.scratch/phase1-base-vs-sft-eval/PRD.md](../../../../.scratch/phase1-base-vs-sft-eval/PRD.md)

---

## Five-file output schema

### `accuracy_summary.json` / `accuracy_summary.csv`

List of rows, one per (checkpoint, source) pair.

| Field | Type | Description |
|---|---|---|
| `checkpoint` | str | User-supplied checkpoint name (e.g. `"base"`) |
| `source` | str | Eval source name (e.g. `"gsm8k"`) |
| `n` | int | Number of examples evaluated |
| `accuracy` | float | Fraction of examples scored correct |
| `parse_success_rate` | float | Fraction where the answer parser extracted a number |
| `generated_tokens` | int | Rectangular position count (compute): `batch_size × new_token_cols` including pad fill after early-stop. Use as the denominator for compute cost. |
| `generated_tokens_decoded` | int | Non-pad token count (content length): positions whose id is not `pad_token_id`. Diverges from `generated_tokens` when sequences finish at different steps. Use as the denominator for content-rate metrics. |
| `elapsed_sec` | float | Wall-clock seconds for this source only |

### `details_<checkpoint>_<source>.csv`

One row per evaluated example.

| Field | Type | Description |
|---|---|---|
| `example_id` | str | Source-assigned example identifier |
| `prompt` | str | The full prompt sent to the model |
| `generated` | str | The raw text generated by the model |
| `gold_answer` | str | The ground-truth answer string |
| `predicted_answer` | str | The answer extracted from `generated` (empty if parse failed) |
| `parse_success` | bool | Whether the parser extracted an answer |
| `is_correct` | bool | Whether `predicted_answer` matches `gold_answer` |

### `run_metadata.json`

Reproducibility record for the run.

| Field | Type | Description |
|---|---|---|
| `device` | str | Friendly hardware name, e.g. `"Tesla T4"` or `"CPU"` |
| `dtype` | str | Torch dtype used for model weights, e.g. `"torch.bfloat16"` |
| `torch_version` | str | PyTorch version string |
| `transformers_version` | str | Transformers version string |
| `cuda_version` | str | CUDA version, or `"N/A"` on CPU |
| `seed` | int | The `--seed` value |
| `eval_n` | int | The `--n` value |
| `git_sha` | str | Short git SHA of the code that ran the eval |
| `generation_settings` | dict | Per-source `{max_new_tokens, batch_size, do_sample}` |
| `checkpoints` | dict | Map of `{name: path}` supplied via `--checkpoints` |

### `cost_summary.json`

Timing and cost record for the full run.

| Field | Type | Description |
|---|---|---|
| `run_name` | str | Final component of `--out-dir` |
| `start_time` | str | ISO-8601 local timestamp when the run started |
| `end_time` | str | ISO-8601 local timestamp when the run ended |
| `elapsed_sec` | float | Wall-clock total (includes model load, I/O); used for billing |
| `generation_seconds` | float | Time spent inside `model.generate()` across all calls |
| `gpu_type` | str | Friendly GPU name (same as `run_metadata.device`) |
| `dtype` | str | Torch dtype string |
| `generated_tokens` | int | Total rectangular position count (compute) across all sources and checkpoints |
| `generated_tokens_decoded` | int | Total non-pad token count (content length) across all sources and checkpoints |
| `tokens_per_second` | float | `generated_tokens / generation_seconds` (rectangular throughput; reflects real forward passes) |
| `estimated_cost_usd` | float or null | Dollar cost if `--gpu-cost-per-hour` was supplied |

## CLI signature

```
python -m finpost.evals.eval_exact \
    --checkpoints NAME=PATH [NAME=PATH ...] \
    --sources SOURCE [SOURCE ...] \
    --n N \
    [--seed SEED] \
    --out-dir DIR \
    [--device {cuda,cpu}] \
    [--gpu-cost-per-hour USD] \
    [--batch-size-gsm8k N] \
    [--batch-size-math N]
```

**Quick example (smoke test with tiny-gpt2 on CPU):**

```bash
python -m finpost.evals.eval_exact \
    --checkpoints tiny=sshleifer/tiny-gpt2 \
    --sources gsm8k --n 4 --seed 42 \
    --out-dir results/evals/smoke \
    --batch-size-gsm8k 2 --device cpu
```

**Notes:**
- Each `PATH` in `--checkpoints` must be an HF Hub model id or a local
  directory in HF format (`config.json` + tokenizer files). Raw finpost
  training checkpoints (containing only `model.safetensors` + `state.pt`)
  must be converted to HF format first.
- `--device` defaults to `cuda` if available, else `cpu`.
- The CLI loads one checkpoint at a time so peak GPU memory is one model.
